## Design Context

### Users
Primary users are developers, technical readers, and authors using a browser extension while reading articles, docs, blog posts, and product pages. They want to quickly understand whether a page has meaningful community discussion around it, and if so, extract the signal without manually searching Hacker News, Reddit, and other sources. The interface should support short, repeatable popup interactions during normal browsing, with minimal friction and no visual noise.

### Brand Personality
Discussed should feel smart, efficient, and trustworthy. The name `discussed.dev` suggests distilled discourse rather than hype: a tool that helps users see what the technical community is saying, then compresses that into useful perspective. The tone should feel calm, sharp, and credible. It should evoke confidence in the quality of the signal, not excitement for its own sake.

### Aesthetic Direction
Design for a browser extension first: compact, legible, and information-dense without feeling cramped. Favor a restrained utility aesthetic over a flashy product-marketing look. Use clean structure, strong typography hierarchy, subtle platform color cues, and small moments of polish rather than decorative visuals. Default to a light-first presentation because the current product is already built around a bright, high-clarity popup and settings UI. If motion is used, keep it purposeful and lightweight.

### Design Principles
1. Insight over chrome. The UI should prioritize discussion signal, synthesis, and relevance ahead of decoration or visual novelty.
2. Calm technical confidence. Visual choices should feel credible, precise, and useful to developers; avoid hype, whimsy, or generic AI-product styling.
3. Compact but breathable. The extension should carry meaningful density inside popup constraints while preserving scanability and clear hierarchy.
4. Restraint over flash. Prefer subtle color, typography, and spacing decisions to create quality; avoid showy effects, loud gradients, and unnecessary motion.
5. Accessible by default. Maintain WCAG AA contrast, clear focus states, keyboard-friendly interactions, and respect reduced-motion preferences whenever animation is introduced.
